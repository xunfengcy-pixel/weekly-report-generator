# 🎯 最简单的 Mac 分发方案

## 给用户的使用说明（一句话版）

> 下载 → 解压 → 双击运行

---

## 📦 你需要做的（打包者）

### 方法一：找一台 Mac 打包（推荐）

1. **把项目文件夹复制到 MacBook**

2. **在 Mac 上打开终端，执行：**
   ```bash
   cd weekly_report_generator
   chmod +x build_mac_app.sh
   ./build_mac_app.sh
   ```

3. **等待 2-3 分钟，打包完成**

4. **把 `dist` 文件夹压缩成 zip**
   ```bash
   cd dist
   zip -r ../周报生成器-Mac版.zip .
   ```

5. **把这个 zip 文件发给用户**

---

### 方法二：使用 GitHub Actions 自动打包（无需 Mac）

如果你会用 GitHub，可以配置自动打包：

1. 把代码推送到 GitHub
2. 配置 GitHub Actions 工作流
3. 每次发布时自动打包 Mac 版本

需要的话我可以帮你写这个配置。

## 👤 用户收到后怎么做

### 使用步骤

1. **下载 `周报生成器-Mac版.zip`**

2. **双击解压**，得到一个文件夹

3. **双击文件夹里的「启动周报生成器.command」**

4. **浏览器自动打开**，开始使用！

---

## ⚠️ 可能遇到的问题

### 问题 1：提示"无法打开，因为无法验证开发者"

**解决方法：**

按住 `Control` 键，点击「启动周报生成器.command」，选择「打开」，然后点击「打开」

或者在终端执行：
```bash
xattr -cr 周报生成器.app
```

### 问题 2：提示"没有权限"

**解决方法：**

在终端执行：
```bash
chmod +x 启动周报生成器.command
```

## 🆚 几种方案对比

| 方案 | 你需要做 | 用户需要做 | 难度 |
|------|----------|------------|------|
| **PyInstaller 打包** | 在 Mac 上运行打包脚本 | 双击运行 | ⭐ 最简单 |
| **Docker 容器** | 写 Dockerfile | 安装 Docker | ⭐⭐ 中等 |
| **源码运行** | 给源码 | 安装 Python + 依赖 | ⭐⭐⭐ 复杂 |
