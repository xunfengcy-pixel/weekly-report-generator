# 周报生成器 - 使用说明

## 项目结构

```
weekly_report_generator/
├── app.py                 # 主应用（Streamlit）
├── config.py             # 配置文件
├── template.html         # HTML模板
├── requirements.txt      # 依赖列表
├── example_data.json     # 示例数据
├── test_generate.py      # 测试脚本
├── test_output.html      # 测试输出
└── README.md            # 说明文档
```

## 快速开始

### 1. 启动应用

```bash
cd weekly_report_generator
streamlit run app.py
```

应用会自动在浏览器中打开（地址：http://localhost:8501）

### 2. 使用步骤

#### 第一步：填写基本信息
- 在左侧边栏选择周报日期
- 输入整体数据（在办案件、新增案件、结案数量、涉外纠纷）
- 填写作者信息

#### 第二步：编辑各板块内容
点击顶部标签切换不同板块：
- **劳动纠纷** - 劳动相关案件
- **基建纠纷** - 基建相关案件
- **竞业限制** - 竞业限制案件
- **其他事项** - 其他类型事项

每个板块包含：
1. **数据概览** - 输入数字统计
2. **摘要** - 整体情况描述（支持HTML）
3. **文档链接** - 添加相关文档URL
4. **案件列表** - 详细案件信息

#### 第三步：添加案件
点击案件卡片，填写：
- **案件标题** - 案件名称
- **负责人** - 从下拉菜单选择（可多选）
- **状态标签** - 可选"新收裁决"、"新增案件"
- **案件进展** - 详细进展描述
- **后续安排** - 后续工作计划

#### 第四步：生成和下载
- **保存数据** - 导出JSON数据备份
- **预览HTML** - 查看生成的HTML代码
- **下载HTML** - 下载完整的HTML文件

## 功能特点

### 1. 可视化表单
- 通过表单填写，无需编写代码
- 实时保存到会话状态

### 2. 负责人管理
- 内置7位负责人头像
- 下拉菜单选择，避免拼写错误
- 支持多选（如"kevin、qimeng"）

### 3. 数据保存
- 支持导出JSON格式
- 便于存档和追溯
- 可导入继续编辑

### 4. 格式统一
- 生成的HTML与模板完全一致
- 自动处理负责人头像
- 支持状态标签

## 数据格式说明

### 案件数据结构
```json
{
  "title": "案件标题",
  "progress": "案件进展（支持换行）",
  "follow_up": "后续安排",
  "owners": "kevin、qimeng",
  "status": "新收裁决"
}
```

### 板块数据结构
```json
{
  "overview": {
    "在办案件": 85,
    "活跃案件": 9
  },
  "summary": "摘要内容（支持HTML）",
  "doc_links": [
    {"name": "链接名称", "url": "链接地址"}
  ],
  "cases": [...]
}
```

## 使用技巧

### 1. 案件进展换行
在案件进展文本框中，使用换行（Enter键）可以将内容分段显示。

### 2. 摘要HTML支持
摘要内容支持HTML标签，如：
- `<a href="#xxx">链接</a>` - 锚点链接
- `<strong>加粗</strong>` - 加粗文字
- `<p style="color: #a3a3a3;">灰色文字</p>` - 灰色文字

### 3. 数据备份
定期点击"保存数据"按钮，将JSON文件保存到本地，以防数据丢失。

### 4. 批量编辑
可以先在JSON文件中批量编辑，然后导入到应用中。

## 部署到云端

### 方案1：Streamlit Cloud（推荐）

1. 将代码推送到GitHub仓库
2. 访问 https://streamlit.io/cloud
3. 使用GitHub账号登录
4. 点击"New app"
5. 选择仓库和分支
6. 设置主文件路径为 `app.py`
7. 点击部署

支持任何支持Python的云平台：
- Heroku
- Railway
- PythonAnywhere
- 阿里云/腾讯云等

## 常见问题

### Q1: 如何修改负责人照片？
编辑 `config.py` 文件中的 `AVATAR_URLS` 字典。

### Q2: 如何添加新的板块？
编辑 `config.py` 文件中的 `SECTIONS` 列表。

### Q3: 如何修改默认作者信息？
编辑 `config.py` 文件中的 `DEFAULT_AUTHOR`。

### Q4: 生成的HTML可以直接发送邮件吗？
可以，生成的HTML文件可以直接作为邮件内容发送。

### Q5: 如何导入之前保存的JSON数据？
当前版本需要手动将JSON数据粘贴到代码中，后续版本会添加导入功能。
