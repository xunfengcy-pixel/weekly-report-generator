# 🚀 用户快速开始指南

## 给 Mac 用户的一句话说明

```
1. 安装 Docker Desktop
2. 双击 start.sh
3. 浏览器自动打开
```

---

## 详细步骤

### 第一步：安装 Docker Desktop

**Mac 用户：**
1. 访问 https://docs.docker.com/desktop/install/mac-install/
2. 下载 Apple Silicon (M1/M2) 或 Intel 版本
3. 双击下载的 `.dmg` 文件，拖拽安装
4. 打开 Docker Desktop，等待启动完成

**Windows 用户：**
1. 访问 https://docs.docker.com/desktop/install/windows-install/
2. 下载并安装
3. 重启电脑后打开 Docker Desktop

---

### 第二步：获取周报生成器

下载并解压 `weekly_report_generator.zip`

文件夹内容：
```
weekly_report_generator/
├── start.sh          # Mac/Linux 启动脚本 ⭐
├── start.bat         # Windows 启动脚本 ⭐
├── docker-compose.yml
├── Dockerfile
├── app.py
├── config.py
├── template.html
└── requirements.txt
```

---

### 第三步：启动应用

#### Mac 用户

1. 打开「终端」应用（按 `Cmd + Space`，输入 Terminal）
2. 输入以下命令：
   ```bash
   cd ~/Downloads/weekly_report_generator
   ./start.sh
   ```
   或者直接把 `start.sh` 拖到终端窗口按回车

3. 等待显示 "✅ 周报生成器启动成功！"
4. 浏览器会自动打开 `http://localhost:8501`

#### Windows 用户

1. 双击 `start.bat`
2. 等待显示 "✅ 周报生成器启动成功！"
3. 浏览器会自动打开

---

## 🛑 停止应用

### Mac
```bash
cd ~/Downloads/weekly_report_generator
docker-compose down
```

### Windows
双击 `stop.bat`（需要创建）或在命令行运行：
```cmd
cd weekly_report_generator
docker-compose down
```

## 💾 数据在哪里？

所有数据保存在 `weekly_report_generator/data/` 文件夹中：
- `weekly_report_autosave.json` - 自动保存的周报数据

**备份方法：** 直接复制 `data` 文件夹

---

## ❓ 常见问题

### Q: 提示 "command not found: docker"
A: Docker Desktop 没有启动，请先打开 Docker Desktop 应用

### Q: 提示 "permission denied"
A: 给脚本添加执行权限：
```bash
chmod +x start.sh
```

### Q: 端口 8501 被占用
A: 修改 `docker-compose.yml` 文件，把 `8501:8501` 改成 `8502:8501`，然后访问 `http://localhost:8502`

### Q: 如何更新到最新版本？
A: 重新下载 zip 文件，替换所有文件（保留 data 文件夹），然后重新运行 start.sh

## 📞 需要帮助？

1. 查看终端输出的错误信息
2. 运行 `docker-compose logs` 查看详细日志
3. 检查 Docker Desktop 是否正常运行
